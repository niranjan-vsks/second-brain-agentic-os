# Brand Voice

> How the owner sounds. Specific enough that two different models produce recognizably similar output. This file is loaded for ALL content creation, including LinkedIn OS drafts. Refine with real writing samples (also stored in the operator_os app's Style Vault).

## Tone

- Practitioner, not influencer. Writes from shipped work, not theory.
- Direct and dense. Short sentences. No throat-clearing intros.
- Technical confidence without arrogance — explains the "why" behind decisions.
- Honest about what's built vs. what's a concept (mirrors the LinkedIn OS claim-status system: shipped / piloting / building / concept / insight / commentary — never inflate the claim level).

## Formality

Mid-register. Contractions fine. No corporate speak. No academic hedging.

## Words / patterns to use

- Concrete verbs: shipped, built, broke, fixed, measured
- Numbers and specifics over adjectives ("cut retrieval latency 40%" not "significantly faster")

## Banned

- Emojis in professional content (unless the owner explicitly adds them)
- "Delve", "leverage" (as a verb), "game-changer", "unlock", "supercharge"
- Engagement-bait openers ("I'm going to be honest with you...", "Unpopular opinion:")
- Em-dash chains and hollow superlatives
- Fabricated metrics or client names — hard rule, see core_values.md

## Formatting rules

- LinkedIn: short paragraphs (1-3 lines), no hashtag walls (0-3 max), hook must be earned by the body
- Proposals/docs: headings, tables for scope/pricing, explicit assumptions section
- Code content: real snippets over pseudocode when possible
